# 프로젝트 계획서

1. 프로젝트 주제

2차원 격자 맵에서 3종류의 전설적인 검을 파밍하고, 총 3층으로 이루어진 사냥탑을 올라가며 5종류의 독특한 몬스터를 사냥하는 1인용 턴제 액션 RPG 게임

2. 문제 상황 및 해결 전략

가. 캐릭터의 2차원 격자 맵 이동 제어 및 아이템 실시간 습득

문제 상황: 플레이어가 W, A, S, D 키를 입력했을 때 12행 20열 격자 맵 범위 밖(돌벽 구역)으로 나가지 않아야 하며, 맵 곳곳에 떨어진 3종류의 검 아이템을 획득할 때 불필요한 키 입력이나 턴 소모가 발생하지 않아야 합니다.

해결 전략: 플레이어의 현재 좌표 (x, y)를 변수로 관리하고 입력 키에 따라 증감 연산을 처리합니다. 새로운 좌표가 외각 벽 영역(가로 0 또는 19, 세로 0 또는 11)을 벗어나려고 하면 이동을 제한하는 예외 처리를 수행합니다. 또한, 플레이어 이동 연산이 일어나는 함수 내부에서 플레이어의 현재 좌표와 아이템 좌표가 일치하는지를 실시간으로 충돌 검사하여, 일치 시 턴 소모 없이 즉시 공격력을 상승시키고 필드에서 아이템을 제거하는 알고리즘을 설계합니다.

나. 몬스터 능력치 모델링 및 플레이어 초기 스탯 하향 밸런싱

문제 상황: 5종류의 서로 다른 몬스터가 등장할 때 일관된 능력치 밸런스를 유지해야 하며, 플레이어의 초기 공격력이 너무 강력하면 아이템 파밍의 필요성이 감소합니다.

해결 전략: 5종류의 몬스터 데이터(이름, 체력, 공격력, x좌표, y좌표, 생존 여부)를 2차원 리스트 내부의 요소로 체계화하고, 체력이 높은 몬스터는 공격력이 약하고 체력이 낮은 몬스터는 강력한 공격력을 가지도록 역비례 모델링을 적용합니다. 또한, 플레이어의 초기 공격력을 15로 낮게 설정함으로써, 사냥터 곳곳을 탐험하며 더 강한 검 아이템을 획득해야만 상위 레벨의 몬스터들을 효과적으로 사냥할 수 있도록 탐험 유도형 기획 밸런싱을 구축합니다.

다. 3층 탑(Floor) 제어 알고리즘 및 플레이어 스탯 연계

문제 상황: 단판으로 끝나는 게임 구조를 탈피하여 다단계 콘텐츠(1층 ~ 3층)를 제공해야 하며, 층이 올라갈 때마다 플레이어가 기존 층에서 강화한 공격력과 아이템 정보를 그대로 유지한 채 체력 회복 등의 승급 보너스를 제공해야 합니다.

해결 전략: 탑의 진행 상태를 나타내는 current_floor 변수와 최대 제한 단계인 max_floors 변수를 선언합니다. 현재 층의 몬스터 5마리를 모두 사냥하면 처치 카운트(kills)를 0으로 초기화하고, 플레이어 변수 자체를 매개변수(existing_player)로 활용하여 기존의 공격력 데이터를 보존한 채 현재 체력만 최대 체력으로 가득 채웁니다. 이후 몬스터와 아이템을 새롭게 재배치하여 층 이동 알고리즘을 매끄럽게 처리합니다.

3. 입력-처리-출력 구조

입력
난이도 입력: 게임 시작 시 사용자로부터 난이도 선택 변수('1': 쉬움, '2': 어려움)를 입력받습니다.
행운의 정수 입력: 어려움 모드 시 난수 생성 시드로 사용할 임의의 정수값(lucky_seed)을 각 층의 시작점마다 입력받습니다.
행동 입력: 플레이어 조작 키 W (위), A (왼쪽), S (아래), D (오른쪽), F (공격), Q (종료)를 문자열로 입력받습니다.

처리
수학적 난수 좌표 도출: 선형 합동법 연산식을 통해 외부 라이브러리 없이 무작위 좌표 세트를 계산합니다.
맵 경계면 검사 및 예외 처리: 이동 방향에 맞춰 다음 좌표를 임시 계산하고 가로 1~18, 세로 1~10 구역 이내일 때만 최종 좌표로 갱신하여 벽 관통 현상을 원천 방지합니다.
턴 소모 없는 검 아이템 자동 장착: 플레이어 좌표가 필드 상에 존재하는 3종 검(기본칼 $+5$, 단검 $+7$, 카람빗 $+10$)의 좌표와 일치할 시, 스탯을 누적 합산하고 필드에서 소멸시킵니다.
전투 연산 및 쉬움 모드 전용 보너스: 플레이어의 사정거리(절댓값 연산을 통해 인접한 1칸 이내)에 있는 몬스터를 타격하여 체력을 깎고 사망 시 처치수를 올립니다. 생존 시 몬스터가 플레이어를 역공합니다. 이때, 쉬움 모드라면 처치 성공 시 플레이어의 체력을 최대 100을 넘지 않는 범위(min 연산 활용) 내에서 즉시 20만큼 회복합니다.
층 승급 연산: 누적 처치 수가 5마리에 도달하면 플레이어 스탯을 전달받아 다음 층으로 리셋 빌드업을 거친 후 current_floor 단계를 올립니다. 3층을 완료하면 최종 승리합니다.

출력
상단 상태 바 표시: 현재 탑의 층 수, 캐릭터 HP 게이지바([■■■■■□□□□□]), 현재 체력/최대 체력, 공격력, 현재 층 사냥 성공 수를 깔끔한 UI로 시각화합니다.
2차원 격자 맵 실시간 출력: 원본 경기장 데이터(map_template)를 가공하여 $12 \times 20$ 격자 공간에 이모지 기호를 맵 형식으로 출력합니다.
플레이어: 🧝
몬스터 5종: 🟢 (슬라임), 🐷 (리본돼지), 🍄 (주황버섯), 🐗 (와일드보어), 😈 (발록의 영혼)
검 아이템 3종: 🗡️
외각 돌벽 테두리: #
안쪽 바닥 빈 구역: .

4. 2차원 리스트 설계
내 프로그램에서 사용할 2차원 리스트의 이름: game_map (뼈대용 템플릿: map_template)

구조 및 크기: 12x20 (12행 20열 격자판)

# map_template = [
#     [0, 0, 0, ..., 0], # row = 0 (외각 테두리 벽 구역)
#     [0, 0, 0, ..., 0], # row = 1 (실제 필드 플레이 구역)
#     ...
#     [0, 0, 0, ..., 0]  # row = 11 (외각 테두리 벽 구역)
# ]

각 행(Row)과 열(Column)의 의미
한 행(Row)의 의미: 맵의 세로 축 좌표(Y축)를 의미합니다. 행 인덱스 0번부터 11번까지 총 12줄이 존재합니다.
각 열(Column)의 의미: 맵의 가로 축 좌표(X축)를 의미합니다. 열 인덱스 0번부터 19번까지 총 20줄이 존재합니다
격자판 안의 원소 값에 따른 상태 매핑:
0: 바닥 빈 땅 (출력 시 테두리는 # , 안쪽은 . 으로 변환)
1: 플레이어 캐릭터 (🧝)
2 ~ 6: 1~5번 몬스터 (🟢, 🐷, 🍄, 🐗, 😈)
7 ~ 9: 3종 검 아이템 (🗡️)
5. 함수 설계

① show_intro()
역할: 게임 실행 시 최초 1회 화면에 타이틀 로고, 키 조작법, 검 아이템 종류별 성능 정보, 난이도별 혜택 및 규칙 등을 깔끔하게 소개합니다.
매개변수 / 반환값: 없음 / 반환값 없음 (단순 UI 텍스트 인쇄 전용)

② get_pseudo_random(seed, min_val, max_val)
역할: 외부 모듈 없이 의사 난수 알고리즘 식을 이용하여 인자로 주어진 seed 값에 수학적 대입 연산을 취해 새로운 범위 내 정수를 생성합니다.
매개변수: 계산의 축이 될 시드값, 범위 최솟값, 범위 최댓값
반환값: (범위 내에서 수학적으로 계산 완료된 랜덤 정수, 다음 계산 시 사용할 새로운 시드값)

③ init_game(difficulty_choice, lucky_seed, existing_player=None)
역할: 게임을 최초 가동하거나 다음 층으로 단계가 진급될 때 플레이어 캐릭터 사전, 몬스터 정보 리스트, 아이템 필드 정보를 통합 정비하여 반환합니다.
매개변수: 선택 난이도, 행운의 번호(난수 시드), 승급 시 전달할 이전 층의 누적 플레이어 딕셔너리
반환값: player(딕셔너리), monsters(2차원 리스트), items(2차원 리스트)

④ draw_screen(player, monsters, items, current_floor)
역할: 맵 템플릿의 데이터를 안전하게 메모리에 새로 복사(깊은 복사)한 뒤, 현재 생존한 캐릭터와 아이템의 좌표를 매핑하여 이중 반복문을 통해 실시간 화면을 그려줍니다.
매개변수: 플레이어 사전, 몬스터 리스트, 아이템 리스트, 현재 탑의 층 정보
반환값: 없음 (가상 격자판 실시간 프린팅 출력)

⑤ move_player(player, key, items)
역할: 사용자가 입력한 WASD에 해당하는 문자열을 좌표 증감으로 변형하여 벽 관통 여부 검사 후 위치를 업데이트하고, 아이템을 밟았을 시 턴 차감 없이 줍기 효과를 실시간으로 입혀줍니다.
매개변수: 플레이어 사전, 가공된 방향 조작 키, 아이템 리스트
반환값: 없음 (플레이어의 딕셔너리 내부 좌표 및 공격력 스탯 데이터 직접 누적 수정)

⑥ check_battle(player, monsters, difficulty_choice)
역할: 플레이어가 근접 사정거리 내에 몬스터가 존재할 때 타격 및 반격 대미지 연산을 적용하고, 사망 연산 및 처치수 변수 카운팅을 수행하며, 쉬움 모드일 경우 처치 HP 보정 회복 조건을 실시간으로 검사합니다.
매개변수: 플레이어 사전, 몬스터 리스트, 현재 선택된 게임 난이도 정보
반환값: 없음 (전투 결과 텍스트 출력 및 체력 차감 연산 반영)

6. 사용할 제어구조

조건문(if-elif-else)을 사용할 부분
난이도 및 대륙 번호 판정: 입력받은 번호가 정상적인 범위의 텍스트가 맞는지 필터링하고 설정 제한 값을 다르게 설정할 때 다중 조건문을 사용합니다.
플레이어 좌표 이동 및 벽 충돌 방지: 예상 이동 좌표가 1~18(가로), 1~10(세로) 이내인 범위 안전 요건을 통과했는지 검사하는 조건식에 사용합니다.
몬스터 및 아이템 중복 생성 방지: LCG를 통해 임의의 좌표가 산출되었을 때, 기존의 플레이어나 다른 몬스터들의 좌표 셋이 누적되어 있는 리스트 내부 값들과 일치하는지를 검출하는 판정에 사용합니다.
절댓값 사정거리 판정 및 전투: 플레이어의 현재 좌표와 몬스터 좌표의 가로 및 세로 절댓값 차이가 각각 1 이하인지를 검출하는 복합 다중 조건 연산에 사용합니다.
체력 및 층수 한계점 예외 방어: 몬스터 사냥 HP 회복 시 최대 HP 상한선 100을 넘지 않게 조절하는 수치 범위 가드 및 3층 돌파 여부 승패 결정 분기에 사용합니다.
반복문(while, for, 중첩 for)을 사용할 부분
while 메인 루프: 플레이어가 생존해 있고 전체 턴 제한 수 이내이며 3층 탑 최종 완료 구문을 격파하지 않는 동안 게임을 계속 가동하는 전체 주 루프에 사용합니다.
while 에러 핸들링 무한 루프: 정상적인 난이도 선택 번호나 무작위 시드 정수값을 사용자가 타자 칠 때까지 올바른 문자를 강제 유도하는 입력 가드 루프에 사용합니다.
for 일괄 정보 매핑 및 탐색: draw_screen 함수 안에서 살아 있는 몬스터들과 필드 상 아이템 좌표를 가상 리스트에 투영하거나, 충돌 대상체를 5마리 몬스터 목록에서 순회 비교할 때 사용합니다.
중첩 for (이중 루프): 12행 20열 격자의 좌표 도화지 칸을 왼쪽 맨 위부터 오른쪽 맨 아래 줄까지 한 칸씩 한 땀 한 땀 탐색하며 이모지 이미지 기호를 그려줄 때 사용합니다.


7. 예상되는 예외 상황 및 해결책

① 지정되지 않은 오타 키보드 오작동 입력
예상 현황: WASDFQ 이외의 엉뚱한 문자나 영문 소문자 등을 입력할 경우 턴 수 카운트가 버려지거나 오류로 꺼질 수 있습니다.
해결 전략: 입력받은 문자열 데이터를 .strip().upper() 내장 함수를 활용해 가공한 뒤 not in ['W', 'A', 'S', 'D', 'F'] 조건 검사를 수행해 정상 키가 아니면 턴 소모 없이 continue를 태워 다시 정상 입력을 유도하도록 방어 시스템을 구축합니다.

② 체력 회복 시 한계점 100 초과 에러 (오버플로우 현황)
예상 현황: 쉬움 모드 처치 HP 회복 보너스 $+20$이 추가될 때, 플레이어가 풀 체력(95 또는 100) 상태라면 최대치 100 한계점을 뚫고 체력이 올라가 구조가 망가집니다.
해결 전략: min() 수학 내장 함수를 활용해 min(player['hp'] + 20, 100)을 계산하여, 어떤 상태에서도 100을 절대 초과하여 대입되지 않도록 예외 처리의 완성도를 높입니다.

③ 2차원 원본 템플릿의 주소값 참조 누적 훼손 현상
