# 프로젝트 계획서

## 1. 프로젝트 주제

내가 만들 프로그램의 주제: 2차원 격자 맵에서 5종류의 독특한 몬스터를 탐색하고 사냥하는 1인용 턴제 액션 RPG 헌터 

## 2. 문제 상황

이 프로그램이 해결하려는 문제를 설명하세요.

가. 캐릭터의 2차원 격자 맵 이동 제어

문제: 플레이어가 W, A, S, D 키를 입력했을 때 격자 안에서 자연스럽게 상, 하, 좌, 우로 한 칸씩 이동해야 하며, 경기장(격자 맵) 밖으로 벽을 뚫고 나가지 않아야 합니다.

해결 전략: 플레이어의 현재 좌표 (player_x, player_y)를 변수로 관리하고, 입력된 키에 따라 좌표 값을 더하거나 뺍니다. 이후, 새로운 좌표가 격자 경계(예: 0 미만 또는 가로/세로 최대 크기 이상)를 벗어나면 원래 위치로 되돌리는 예외 처리를 수행합니다.

나. 몬스터들의 차별화된 능력치 모델링 (HP와 공격력의 반비례 관계)

문제: 5종류의 서로 다른 몬스터가 등장해야 하며, 밸런스를 위해 HP가 높을수록 공격력은 낮고 HP가 낮을수록 공격력은 높은 개성 있는 능력치를 설정해야 합니다.

해결 전략: 5종류의 몬스터 데이터(이름, HP, 공격력, 현재 위치, 살아있는지 여부)를 2차원 리스트 또는 딕셔너리가 결합된 형태로 모델링합니다.
예를 들어 monsters = [["슬라임", 30, 15, x, y], ["리본돼지", 50, 10, x, y], ...]와 같이 설정하여 데이터 간의 일관성을 유지하고, 캐릭터가 해당 위치에 도달했을 때 적절한 전투가 일어나도록 설계합니다.

다. 격자 맵 상에서의 충돌(전투) 판정 및 턴제 전투

문제: 플레이어가 몬스터와 같은 칸에 위치하거나 인접한 칸에 있을 때 어떻게 전투를 치르고 HP를 깎을 것인지 연산해야 합니다.

해결 전략: 플레이어와 각 몬스터의 가로, 세로 좌표 차이를 연산합니다. 플레이어가 공격 키(F)를 입력했을 때, 몬스터가 플레이어의 사정거리(동일한 칸 또는 1칸 인접) 안에 있다면 공격을 성공시키고 몬스터의 HP를 감소시킵니다. 몬스터가 살아있다면 반격(플레이어 HP 차감)이 진행되도록 턴을 설계합니다.

## 3. 입력-처리-출력 구조

### 입력

행동 입력: 플레이어 조작 키 W (위), A (왼쪽), S (아래), D (오른쪽), F (공격)를 입력받습니다.

기타 입력: 게임 시작 여부(Y/N) 또는 게임 종료 후 재시작 여부를 입력받습니다.
### 처리

좌표 업데이트: 입력받은 방향키(WASD)에 따라 플레이어의 다음 좌표를 임시 계산합니다.

맵 경계면 검사: 플레이어가 맵의 경계(예: 가로 0~14, 세로 0~7)를 벗어나지 못하도록 이동 범위를 제한합니다.

전투 연산 (공격 및 반격):

플레이어가 F(공격) 키를 누르면 사정거리(상하좌우 1칸 또는 같은 칸)에 있는 몬스터를 찾아 공격을 수행하고 몬스터의 HP를 깎습니다.

살아남은 몬스터는 플레이어를 반격하여 플레이어의 HP를 몬스터의 공격력만큼 차감합니다.

몬스터 상태 업데이트: HP가 0 이하가 된 몬스터는 맵에서 사라지고 경험치(또는 점수)가 누적됩니다.

게임 종료 조건 검사:

플레이어의 HP가 0 이하가 되면 '패배(Game Over)' 처리합니다.

맵 안의 5마리 몬스터를 모두 잡으면 '승리(Clear)' 처리합니다.

### 출력

상태 바 표시: 플레이어의 현재 HP, 공격력, 잡은 몬스터 수, 현재 처치 대상 정보를 상단에 텍스트와 게이지(예: HP: [■■■■□] 80/100)로 보여줍니다.

2차원 격자 맵 출력: 8 x 15 크기의 격자를 콘솔에 시각화하여 출력합니다.

플레이어: P (또는 🧝)

슬라임, 리본돼지, 주황버섯 등 몬스터들: 각각 고유 기호 (예: 1, 2, 3, 4, 5 또는 이모지)

빈 공간: .

격자 경계 벽면: #

### 한 행의 의미
게임 맵의 가로 줄(Y축 위치)을 의미합니다. 인덱스 0부터 7까지 총 8개의 행이 존재하며, 윗줄부터 아랫줄까지의 위치를 나타냅니다.

### 각 열의 의미
게임 맵의 세로 줄(X축 위치)을 의미합니다. 인덱스 0부터 14까지 총 15개의 열이 존재하며, 왼쪽 끝에서 오른쪽 끝까지의 위치를 나타냅니다.

각 칸에는 다음과 같은 숫자를 채워 상태를 구분합니다.

0: 빈 공간

1: 플레이어 위치

2 ~ 6: 각각 1성부터 5성까지의 서로 다른 몬스터들의 위치

## 5. 함수 설계

함수 이름: init_game()

역할: 게임 시작 시 플레이어의 기본 정보(HP, ATK, 좌표)와 5마리의 몬스터 종류별 초기 정보(이름, HP, ATK, 좌표, 생존 여부)를 설정하여 반환합니다.

입력값: 없음

반환값 또는 출력: 플레이어 상태 정보(딕셔너리), 몬스터들 목록(2차원 리스트 또는 리스트 내부 딕셔너리)

함수 이름: move_player(player, key, max_rows, max_cols)

역할: 사용자가 입력한 WASD 키에 맞게 플레이어의 가상 좌표를 계산하고 경계를 벗어나지 않는지 예외 처리를 한 후, 안전하게 이동을 반영합니다.

입력값: 플레이어 정보 player, 입력한 키 key, 맵의 세로 크기 max_rows, 가로 크기 max_cols

반환값 또는 출력: 없음 (플레이어의 좌표 값을 직접 갱신)

함수 이름: check_battle(player, monsters, key)

역할: 플레이어가 공격 키(F)를 입력했을 때, 플레이어 주변(같은 칸 또는 인접한 칸)에 살아있는 몬스터가 있는지 확인하여 전투를 진행합니다. 서로 대미지를 주고받는 연산을 수행합니다.

입력값: 플레이어 정보 player, 몬스터 리스트 monsters, 입력한 키 key

반환값 또는 출력: 전투 발생 여부 및 결과 메시지(스트링)

함수 이름: draw_screen(player, monsters, rows, cols)

역할: 플레이어와 몬스터들의 좌표를 읽어와 2차원 리스트 game_map에 임시 배치한 후, 현재 플레이어의 스탯(HP, 공격력, 잡은 마리 수)과 격자 맵을 콘솔 화면에 깔끔하게 그려줍니다.

입력값: 플레이어 정보 player, 몬스터 리스트 monsters, 세로 크기 rows, 가로 크기 cols

반환값 또는 출력: 없음 (화면에 UI와 맵 직접 출력)

## 6. 사용할 제어구조

### 조건문을 사용할 부분
입력된 조작 키가 W, A, S, D, F 중 어느 것인지 판별할 때 사용합니다.

플레이어가 벽 밖으로 넘어가지 않도록 좌표 범위 검사(if x < 0 or x >= 15) 시 사용합니다.

플레이어가 공격을 시도할 때, 플레이어와 몬스터의 거리가 사정거리 이내인지 판단하는 거리 판정 공식에 다중 조건문을 사용합니다.

각 몬스터의 HP가 0 이하가 되었을 때 사망 상태로 전환하는 판정에 사용합니다.

### 반복문을 사용할 부분

while 루프: 게임이 종료 조건(플레이어 사망 혹은 모든 몬스터 처치)을 충족할 때까지 매 턴 키 입력을 받고 화면을 새로 그리는 전체 게임 루프를 유지하는 데 사용합니다.

중첩 for 루프: draw_screen 함수 내부에서 8행 15열의 2차원 리스트 경기장을 탐색하며 빈 공간(.), 플레이어(P), 몬스터(1~5)를 화면에 그려줄 때 사용합니다.

for 루프: 플레이어 주위에 공격 가능한 몬스터가 있는지 5마리의 몬스터 목록을 순회하며 하나씩 좌표를 비교해 볼 때 사용합니다.

## 7. 예상되는 예외 상황

1. 사용자가 WASD 나 F가 아닌 다른 문자나 숫자 키를 입력할 수 있습니다.

이 경우, 좌표를 변경하거나 턴을 소모하지 않고 "올바른 키를 입력해주세요!"라는 에러 경고 문구를 출력한 후 다시 입력을 받도록 예외 처리합니다.

2. 이미 처치하여 HP가 0이 되어 사라진 몬스터의 좌표 자리에 플레이어가 가서 공격(F)을 누를 경우, 죽은 대상에게 데미지가 누적되거나 승리 판정 카운트가 중복으로 올라갈 수 있습니다.

전투 처리 루틴에서 해당 몬스터의 '생존 여부' 상태 값이 True인 경우에만 타격 대상을 선별하도록 필터링 조건을 추가합니다.
